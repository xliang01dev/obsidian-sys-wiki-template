# maintenance index

This folder stores dated wiki health reports and repair logs.
Create one file per run and keep this index updated.

## Naming convention

- `YYYY-MM-DD-lint.md` — structural and semantic audit run
- `YYYY-MM-DD-repair.md` — repair pass after review or lint

## Report workflow

1. Use `templates/template-maintenance.md`
2. Create a new dated file in this folder
3. Fill in the report
4. Add a link and one-line summary below
