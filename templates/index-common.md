# Index Common

Use this file as the shared standard for folder-level `index.md` files.
Folder indexes should stay short, navigable, and specific to the folder they describe.

## Shared rules

- Explain the purpose of the folder in 1 to 3 short paragraphs or bullets.
- List what belongs in the folder and what does not.
- Prefer links, examples, and short bullet lists over long prose.
- Focus on local navigation, not vault-wide rules.
- Update the folder index when files, subfolders, or naming patterns change materially.
- Link to important notes, subfolders, and canonical pages when they exist.
- Avoid repeating detailed global rules already covered in `CLAUDE.md`.
- Avoid duplicating this shared guidance inside every folder index.

## Recommended structure

A folder `index.md` should usually contain:
1. folder purpose
2. what belongs here
3. local rules or constraints
4. key contents, subfolders, or templates
5. brief workflow notes if needed

## Naming and scope

- Use `index.md` as the filename inside each folder.
- Keep the content specific to that folder.
- If a folder grows large, split navigation into sub-indexes rather than turning one index into a long document.
